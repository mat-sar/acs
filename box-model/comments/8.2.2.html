Annotated CSS Spec ?
This comment relates to the following words from Section 8.2 of CSS 2.2:
The example HTML document: […] results in a document tree with (among other relationships) a UL element that has two LI children.

Section 3.1 defines the document tree as [t]he tree of elements encoded in the source document. Each element in this tree has exactly one parent, with the exception of the root element, which has none.

Here is a representation of the document tree which results from the example HTML document:

  • HTML
    • HEAD
      • TITLE
      • STYLE type="text/css"
    • BODY
      • UL
        • LI
        • LI class="withborder"

Terminological remark

The CSS spec defines document tree as a tree of elements. Note that there exists another definition: a tree of nodes. That other definition is broader: a node can be an element, but also a comment, a sequence of text, and others.

The more general definition of document tree is the basis of the Document Object Model, a standard on how to programmatically interact with documents such as HTML. For comparison, here is the example HTML again, this time represented as a tree of nodes:

  • Document
    • Doctype: html
    • Element: HTML
      • Element: HEAD
        • Text: ⏎␣␣␣␣
        • Element: TITLE
          • Text: Examples of margins, padding, and borders
        • Text: ⏎␣␣␣␣
        • Element: STYLE type="text/css"
          • Text: UL { background: yellow; … }
        • Text: ⏎␣␣
      • Text: ⏎␣␣
      • Element: BODY
        • Text: ⏎␣␣␣␣
        • Element: UL
          • Text: ⏎␣␣␣␣␣␣
          • Element: LI
            • Text: First element of list⏎␣␣␣␣␣␣
          • Element: LI class="withborder"
            • Text: Second element of list is⏎␣␣␣␣␣␣␣␣␣␣␣a bit longer to illustrate wrapping.⏎␣␣␣␣
        • Text: ⏎␣␣⏎

As we can see, CSS adopts a more “compact” interpretation of the document tree.

By the way, you might notice that some whitespace present in the source document is not represented in the tree above. For example, the tree has no whitespace between <html> and <head>. This effect follows from the HTML parsing rules.
