Annotated CSS Spec ?
This comment relates to the following words from Section 8.1 of CSS 2.2:
The four content edges define the box's content box.

This fragment of the spec defines a new term, content box. A few sentences earlier, the spec introduces the content area (Each box has a content area (e.g., text, an image, etc.).) How does the content box differ from the content area?

The difference is nuanced:

  • content box: the rectangle delineated by the four content edges (top, right, bottom, left)
  • content area: the area occupied by the content box

Content box and content area have slightly different definitions, but we can use them to point to the same thing. So, having both terms might seem redundant. There is a reason for keeping them, though. While content box and content area might point to the same thing, padding box and padding area do not. Similarly, there is a difference between a border box and a border area, and between a margin box and a margin area.

I address these differences in the subsequent comments.
