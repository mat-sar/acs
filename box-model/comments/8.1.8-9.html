Annotated CSS Spec ?
This comment relates to the following words from Section 8.1 of CSS 2.2:
If the padding has 0 width, the padding edge is the same as the content edge. The four padding edges define the box's padding box.

Let me make a terminological distinction. In CSS, when we say “box”, we typically mean a box created from an element, e.g. from <div> or <img>. In this comment, I will call such a box a “CSS box”. A CSS box is itself composed of boxes: a content box, a padding box, a border box and a margin box.

Here, we focus on the padding box. It is composed of four padding edges: top, right, bottom, left. Importantly, these edges are present in every CSS box, even if we set padding: 0. In this case, the padding edges will be the same as the content edges.

So, every CSS box contains a padding box. In contrast, not every CSS box contains a padding area. This is stated by the spec itself, which calls the padding area optional.

The padding area is the area which results from subtracting the surface occupied by the padding box from the surface occupied by the content box.
