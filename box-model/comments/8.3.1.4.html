Annotated CSS Spec ?
This comment relates to the following words from Section 8.3.1 of CSS 2.2:
Margins of the root element's box do not collapse.

I recommend that you read this comment after you have acquainted yourself with the other comments about margin collapse.

In HTML, the root element is <html>. If we set a vertical margin on the root, this margin will never collapse with any other margin.

Suppose that we set a top margin of 10 pixels on <html> and a top margin of 10 pixels on <body>. These two margins will never collapse.

Similarly, if we have a page in which <html> has a top and a bottom margin and the <body> is completely empty, then the top and the bottom margin will not collapse.

Note that this part of the spec is more of a confirmation of expected behaviour than a new rule. The reason is that in Section 9.3, the spec defines the root element to be an out-of-flow element. This is key because in the section we are discussing now, the spec says that two margins are adjoining if and only if (emphasis mine) both belong to in-flow block-level boxesâ€¦. So, the margins of the root element can never be considered adjoining.
