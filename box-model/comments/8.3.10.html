Annotated CSS Spec ?
This comment relates to the following words from Section 8.3 of CSS 2.2:
Negative values for margin properties are allowed, but there may be implementation-specific limits.

It’s easy to imagine that the four building blocks of a box: content box, padding box, border box and margin box are like Russian dolls, one nested within the other. However, with negative margin values, this analogy breaks to some degree:

Boxes are typically laid out such that the margin edge of one box “touches” the margin edge of another box. Thus, negative margins will often lead to the visual effect of boxes overlapping:

Note that in theory, the overlap effect might be absent in some browsers. The spec states that while negative margins values are valid, there may be implementation-specific limits. That is, a browser might not be able to render a box with a negative margin, and still conform to the CSS 2 spec. In practice, major browsers have supported negative margins for a long time. Already in 2004, Eric A. Meyer wrote in Cascading Style Sheets: The Definitive Guide: As of this writing, there are few, if any, such limits in current browsers.

It was quite on purpose that for the example above, I chose horizontal margins. Vertical margins have the added effect that they can collapse, which adds another level of complexity.
